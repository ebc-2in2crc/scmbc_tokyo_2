branch

ブランチの一覧表示・作成・削除・名前変更を行います。
git branch
git branch (-a|--all)
ブランチ一覧を取得します。
ブランチ名の先頭に「*」が表示されているブランチが、現在作業中のブランチとなる。
git branch --list <pattern>
ブランチの一覧を取得します。
上の２つとの違い、リモートで追跡しているブランチも一覧として取得されます。
また、[pattern]で指定したパターンにマッチするブランチする一覧を表示できます。
git branch (-r|--remote)
リモートで追跡しているブランチの一覧を取得します。
ローカルのブランチは表示されません。
git branch <branchname> <start-point>
ブランチの作成。 <branchname>は作成するブランチ名
作成されたブランチは、現在作業中のブランチのHEADがコピーされます。
また、[start-option]を指定した場合は、指定したコミットを起点としたブランチが作成されます
git branch (-d|-D) <branchname>
ローカル上のブランチを削除します。
リモートのブランチを削除するには git push :branchname として
ブランチ名の前に「:」をつける事で削除できます。
git branch (-m|-M|--move) <oldname> <newname>
ブランチ名を変更します。
引数で指定した[oldname]のブランチ名を[newname]へ変更します。

init

git-init - 空Gitリポジトリを作成 または 既存のものを再初期化します
git init [-q | --quiet] [--bare] [--template=] [--separate-git-dir ] [--shared[=]] [directory]
-q, --quiet 警告とエラーのみを表示します．他のメッセージは破棄します．

--bare
ベアリポジトリを作成します．
ベアリポジトリは作業コピーを持たないリポジトリで，作業を中継するリポジトリとして使用します．

--template=
テンプレートディレクトリを設定します
init時に任意のフックスクリプトをコピーしたい場合に使用します

--separate-git-dir=
任意のディレクトリにリポジトリを設定します
リポジトリとワークツリーを分けたいときに使用します
--shared[=(false|true|umask|group|all|world|everybody|0xxx)] 複数ユーザーでリポジトリを共有する際のアクセス権を設定します．

reset

ワークツリーやインデックスを指定コミットの状態にする
git reset [--soft | --mixed | --hard] []

--soft コミットだけ取り消す
--mixed インデックスの変更を取り消す
--hard ワークツリーとインデックスを破棄して指定コミットの状態にする

status

ワーキングツリーの状態を表示する
git status [<options>..] [--] [<pathspec>...]

--short 簡易表記で表示
--branch ブランチとトラック情報を簡易表記で表示
--porcelain スクリプトとかでパースしやすい形式で表示
--untracked-files[=mode] 未トラックのファイルも表示
  modeの説明
  no 未トラックファイルを表示しない
  normal 未トラックファイルとディレクトリーを表示
  all 未トラックフォルダーの不可視ファイルを表示
--ignore-submodules[=when] サブモジュールの変更を表示しない
--ignored 無視ファイルも表示
-z 改行をLF(0x0a)ではなくNUL(0x00)で表示する

grep

リポジトリに対して特定の語句が含まれているファイルを検索する
git grep [<options>] [<keyword>]

-i / --ignore-case : 大文字小文字を区別せずマッチします
-I : バイナリファイルを無視します
-w / --word-regexp : 検索文字が単語としてマッチする時のみマッチするようになります。このオプションが指定されているとき a は abc にマッチしません
-v / --invert-match : 実際にマッチした単語がない行も表示します
-h / -H : マッチしたファイル名を行頭に表示するか否かの指定です。
--full-name : ファイル名を表示する時、詳細なファイルパスを出力するようにします(デフォルトでは basename のみ)
-E / --extended-regexp : 検索に POSIX の拡張正規表現を利用します
-G / --basic-regexp : 検索に POSIX の標準正規表現を利用します
-P / --perl-regexp : 検索に Perl の正規表現を利用します
-F / --fixed-strings : 検索に正規表現を利用しません
-n / --line-number : マッチしたファイル名の後ろにマッチした行数を表示します
-l / --files-with-matches / --name-only : マッチしたファイルのファイル名だけを表示します
-L / --files-without-match : マッチしなかったファイル名だけを表示します

merge

他のブランチの変更内容をマージする
git merge [-n] [--stat] [--no-commit] [--squash] [-s ] [-X ]
[--[no-]rerere-autoupdate] [-m ] [...]
git merge HEAD ...
git merge --abort
--no-commit マージしたコミットを自動的にコミットしません.
--ff マージするブランチ間がFast-Forwardの関係の場合，コミットを作成しません.
--no-ff マージするブランチ間がFast-Forwardの関係の場合でもコミットを作成します
--squash マージコミットを作成せず，インデックスとワークフローのみ更新します．
